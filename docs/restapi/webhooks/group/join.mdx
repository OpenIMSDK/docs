# 在用户加入群组之前回调

## 功能说明
App 业务服务端可以通过该回调获取用户加入群组的请求，同时业务服务端可以拒绝用户加入群组的请求。

## 注意事项
- 为启用回调，必须配置回调 URL，并开启本条回调协议对应的开关。配置方法详见 [回调说明](../introduction) 文档。
- 回调的方向是OpenIMServer向 App 后台发起 HTTP/HTTPS POST 请求。
- APP 业务服务端需在超时时间内响应此请求。

## 可能触发该回调的场景
- App 用户通过客户端加入群组

## 回调发生时机
- OpenIMServer准备将用户加入群组之前。

## 接口说明

### 请求 URL 示例
以下示例中 App 配置的回调 URL 为 `https://callbackurl`。
```plaintext
https://callbackurl?command=$CallbackCommand
```

### 请求参数说明

| 参数            | 说明                                                   |
| --------------- | ------------------------------------------------------ |
| https          | 请求协议为 HTTPS，请求方式为 POST                      |
| https://callbackurl | 回调 URL                                               |
| CallbackCommand | 固定为：callbackBeforeCreateGroupCommand                |



### 请求包示例
```json
{
  "callbackCommand": "CallbackBeforeMemberJoinGroupCommand",
  "operationID": "1234",
  "groupID": "12345",
  "groupName": "MyGroup",
  "userID": "user789",
  "ex": "Extra data",
  "groupEx": "GroupExtra data"
}

```




### 请求包字段说明

| 对象                   | 类型   | 描述                                                    |
|-------------------------|--------|----------------------------------------------------------------|
| callbackCommand         | string | 回调命令，这里是创建群组之前的回调。                             |
| operationID             | string | 操作的唯一标识符。                                               |
| userID                  | string | 加入群组的用户ID。                                             |
| groupID                 | string | 群组的唯一标识符。                                               |
| groupName               | string | 群组的名称。                                                     |
| ex                      | string | 额外的数据字段。                                                 |
| groupEx                 | string | 群组额外的数据字段。                                              |

## 应答包示例

### 允许创建
允许用户加入群组。

```json
{
    "callbackCommand": "CallbackBeforeMemberJoinGroupCommand",
    "operationID": "1234",
    "groupID": "12345",
    "userID": "user789",
    "nickName": "nickName",
    "faceURL": "url",
    "roleLevel": "1",
    "ex": "Some extra data"
}

```

## 应答包字段说明

| 字段        | 值               | 说明                                       |
|---------------------|---------|------------------------------------------|
| callbackCommand     | string  | 回调命令，这里是创建群组之前的回调。      |
| operationID         | string  | 操作的唯一标识符。|
| groupID             | string  | 群组的唯一标识符。|
| userID              | string  | 加入群组的用户ID。|
| nickName            | string  | 加入群组的用户呢称。|
| faceURL             | string  | 加入群组用户的图标或者图片URL。                         |
| roleLevel           | string  | 加入群组的用户权限                             |
| ex                  | string  | 额外的数据或信息。                         |
