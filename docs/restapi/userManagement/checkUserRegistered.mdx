## 查询用户是否注册

</center>

### 简要描述
- 查询userID是否已经在IM系统中注册，应在IM注册之前调用，确保userID不会重复注册。
### 请求方式
- `post` 
### 请求URL
- `{{host}}/user/account_check` 


### Header

|:----    |:-------    |:--- |---|------      |
|operationID|879654321|必填|string|operationID用于全局链路追踪|

### 请求参数示例


```json
{
  "checkUserIDs": [
    "11111111",
    "11111112"
  ]
}
```
|字段名|选填|类型|说明|
|:----    |:-------    |:--- |---|
|checkUserIDs|必填|array|要查询的用户ID列表|
### 成功返回示例


```json
{
  "errCode": 0,
  "errMsg": "",
  "errDlt": "",
  "data": {
    "results": [
      {
        "userID": "11111111",
        "accountStatus": "registered"
      },
      {
        "userID": "11111112",
        "accountStatus": "registered"
      }
    ]
  }
}
```
### 成功返回示例的参数说明


|参数名|类型|说明|
|:----    |:-------    |:--- |
|errCode|int|错误码,0表示成功|
|errMsg|string|错误简要信息,无错误时为空|
|errDlt|errDlt|错误详细信息,无错误时为空|
|data|object|通用数据对象，具体结构见下方|
|results|array|查询结果|
|results.userID|string|用户ID|
|results.accountStatus|string|账户状态，注册或未注册|
### 失败返回示例


```json
{
  "errCode": 1001,
  "errMsg": "ArgsError",
  "errDlt": "userID repeated[1001]ArgsError"
}
```
### 失败返回示例的参数说明


|参数名|类型|说明|
|:----    |:-------    |:--- |
|errCode|int|错误码,具体查看全局错误码文档|
|errMsg|string|错误简要信息|
|errDlt|errDlt|错误详细信息|
