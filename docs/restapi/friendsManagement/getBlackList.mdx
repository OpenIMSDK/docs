## 获取用户黑名单列表

</center>

### 简要描述
- 获取用户黑名单列表，该接口需要使用管理员token，支持分页拉取
### 请求方式
- `post` 
### 请求URL
- `{{host}}/friend/get_black_list`

### 请求参数示例


```json
{
  "userID": "11111111",
  "pagination": {
    "pageNumber": 1,
    "showNumber": 10
  }
}
```
|字段名|选填|类型|说明|
|:----    |:-------    |:--- |---|
|userID|必填|string|用户ID|
|pagination|必填|object|分页结构体|
|pagination.pageNumber|必填|string|分页号|
|pagination.showNumber|必填|string|每个分页显示用户数量|

```json
          "ex": ""
        },
        "addSource": 0,
        "operatorUserID": "openIMAdmin",
        "ex": ""
      }
    ],
    "total": 1
  }

```
### 成功返回示例的参数说明


|参数名|类型|说明|
|:----    |:-------    |:--- |
|errCode|int|错误码,0表示成功|
|errMsg|string|错误简要信息,无错误时为空|
|errDlt|errDlt|错误详细信息,无错误时为空|
|data|object|通用数据对象，具体结构见下方|
|blacks|array|黑名单列表|
|blacks.ownerUserID|string|用户ID|
|blacks.createTime|string|拉黑时间|
|blacks.blackUserInfo|object|拉黑的用户信息|
|blacks.blackUserInfo.userID|string|拉黑的用户ID|
|blacks.blackUserInfo.nickname|string|拉黑的用户昵称|
|blacks.blackUserInfo.faceURL|string|拉黑的用户头像|
|blacks.blackUserInfo.ex|string|拉黑的用户ex字段|
|blacks.addSource|int|拉黑来源|
|blacks.operatorUserID|string|操作者ID|
|blacks.ex|string|ex字段|
|total|string|总数|
### 失败返回示例


```json
{
  "errCode": 1004,
  "errMsg": "RecordNotFoundError",
  "errDlt": "[1004]RecordNotFoundError"
}
```
### 失败返回示例的参数说明


|参数名|类型|说明|
|:----    |:-------    |:--- |
|errCode|int|错误码,具体查看全局错误码文档|
|errMsg|string|错误简要信息|
|errDlt|errDlt|错误详细信息|
